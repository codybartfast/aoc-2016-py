# Advent of Code 2016 in Python

Hacking EBHQ for Santa in the timely [Advent of Code 2016](https://adventofcode.com/2016)

Launched in 2026, this is my second Advent 2016 campaign.

The [previous attempt](https://git.sr.ht/~codybartfast/aoc-2016-fs) was in F#.  I was pleased that most of these solutions  
run faster than the F# ones but some of that is probably just down to the F#  
startup time.  Also, I was often focusing on performance this time but was more  
concerned with coding time in the F# project.  

The Day 5 solution is probably more efficient than many of my previous solutions  
to hash puzzles.

Day 11 looked a lot simpler than it was so I wasted time on failed attempts at  
a more insightful solution before resorting to brute force.

I also made a bit of a Horlicks out of Day 19 but it was fun detour.  

In contrast to Day 11, I think Day 22 hinges on realising it's probably a lot  
simpler than it appears.  A key mistake with AoC can be committing too early to  
how hard one thinks the solution is.

*`Python ver: 3.14.2,  Timings: MacBook M4`*
